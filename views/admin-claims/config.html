Admin Claim Configuration

Configure admin claim types and their protection settings.

Admin Claim Types

Admin claims are organized by types, each with its own configuration file. You can set any type name you want such as spawn, wilderness, pvp-arena, shopping-district or any other custom type needed for your server.

Configuration Files

Each admin claim type has its own configuration file located at:

/config/claimorous_admin_claims/<type>.toml

For example, the "spawn" claim type configuration would be at /config/claimorous_admin_claims/spawn.toml.

Creating New Admin Claim Types

To create a new admin claim type:

  1. Use the command: /claimorous admin config <new_type_name>
  2. Configure the settings in the GUI that appears
  3. Save the configuration

Alternatively, you can copy an existing type's TOML in the proper path or share them on Discord!

Example Configurations

Spawn HUB

An example spawn area configuration might look like this: everything is protected except for chest, beds and doors. Book could also be used to read useful information.

 
# [General Settings]

# Claim Color
claimColor = "#891E26"
# Claim Border
showClaimBorder = true
# Permission System
enablePermissionSystem = true

# [Block and Item Protection Settings]

preventBlockBreaking = true
preventBlockBreakingExceptionsIds = []
preventBlockBreakingExceptionsTags = []

preventBlockPlacement = true
preventBlockPlacementExceptionsIds = []
preventBlockPlacementExceptionsTags = []

preventBlockInteraction = true
preventBlockInteractionExceptionsIds = [minecraft:chest]
preventBlockInteractionExceptionsTags = [minecraft:doors, minecraft:beds]

preventItemUse = true
preventItemUseExceptionsIds = [minecraft:book]
preventItemUseExceptionsTags = []


# [PvP and Projectiles Protection Settings]

# PvP Protection
protectEveryonePvP = true

# Projectiles Protection
preventProjectileDamage = true
showProjectileEffects = true


# [Entity Protection Settings]
entityProtectedFromAttack = true
entityDamageProtectionExceptionsIds = []
entityDamageProtectionExceptionsTags = []
entityProtectedFromInteraction = true
entityInteractionProtectionExceptionsIds = []
entityInteractionProtectionExceptionsTags = []
protectFromHostileMob = true


# [Environment Protection Settings]

preventFluidFlowIntoClaim = true

# Fire Protection
preventFireSpreadAcrossClaim = true

# Explosion Protection
preventExplosion = true
showExplosionEffects = true

# Piston Protection
preventPistonInteraction = true

# Dispenser Protection
preventDispenserInteraction = true

Important

Changes to admin claim configurations take effect immediately without requiring a server restart. However, it's recommended to make changes when there are no players online to avoid disruption.

Importing and Exporting

You can easily share admin claim configurations between servers by copying the TOML files. This is useful for maintaining consistent settings across multiple servers or for backup purposes.

To import a configuration:

  1. Copy the TOML file to the /config/claimorous_admin_claims/ directory
  2. You might need to restart your game.
